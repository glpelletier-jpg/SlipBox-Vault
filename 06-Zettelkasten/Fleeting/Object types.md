# Object types

An [object](https://docs.capacities.io/reference/glossary#object) is the fundamental unit of content in Capacities. Every [object](https://docs.capacities.io/reference/glossary#object) in Capacities has a type. It tells you the **type of information stored in the note**.

![image](../Images/Media/image%20(13).png)
[image - Notes](../Images/image%20(13).md)

All your object types live in the left sidebar. All your objects can be found within one of them, because your objects are always an instance of one of the object types.

![image](../Images/Media/image%20(7).png)
[image - Notes](../Images/image%20(7).md)

Object types are either created automatically by Capacities, or you can create them yourself. The first group is what we call Basic Object Types, and the second are Custom Object Types. Their settings define the structure of the content. Read on to see how this works.

## [Basic object types](https://docs.capacities.io/reference/content-types#basic-object-types)

Basic object types are types that are built into the app. Each user has access to them.

Basic types are designed to be useful to all users and to cover the most basic use-cases first. Since basic object types are built into the app, we can give them a tailored design and functionality. We can make them more powerful with custom functionality. That’s their big strength.

Here are all basic object types in Capacities:

- **Page**

    This is the simplest object type for writing notes. Everything is set up for you, all you need to do is start typing!

    Read more about the pages [here](https://docs.capacities.io/reference/basic-types/pages).

- **Tag**

    Tags are a very useful organization object type in Capacities. Think of them like keywords that you can assign objects or blocks. If you click on a tag, you can see all of the places it has been used.

    Read more about tags [here](https://docs.capacities.io/tutorials/tags).

- **Image**

    Any image you input into Capacities becomes an image object, that can be reused anywhere in your space.

    Read more about images [here](https://docs.capacities.io/reference/basic-types/images).

- **Weblink**

    Any links you paste into Capacities will be recognized as a link, and you will have the option to create a weblink from them, or to keep the plain URL. The one difference is that the URL is just a link, whereas the weblink is an object that gains all the functionality of other objects with some extra features.

    Read more about weblinks [here](https://docs.capacities.io/reference/basic-types/weblinks).

- **Audio**

    You can save audio files to Capacities via upload, Whatsapp or Telegram.

    Read more about these [here](https://docs.capacities.io/reference/basic-types/audio).

- **PDF**

    You can upload PDFs, read them and tag them in Capacities so they become part of your notes.

    Read all about this [here](https://docs.capacities.io/reference/basic-types/pdfs).

- **Files**

    You can store files of various types in Capacities. This allows you to integrate them with your notes by referencing them or tagging them.

    You can read more about them [here](https://docs.capacities.io/reference/basic-types/files).

- **Tweet**

    Whilst the full Twitter/X integration is disabled until further notice due to API changes, you can still save URLs of Tweets to Capacities. Every time you do this, the link will automatically become a Tweet object.

    You can read more about this [here](https://docs.capacities.io/reference/integrations/twitter).

- **AI Chat**

    Sometimes you might want to come back to chats you've had with the AI assistant. Saving them as objects allows you to do that.

    See more [here](https://docs.capacities.io/reference/ai-assistant#saving-ai-chats-as-objects).

- **Query**

    Queries collect content based on rules. All queries you create are saved as objects for you to reuse in your capacities space.

    See more about [queries on their page](https://docs.capacities.io/reference/queries).

- **Table**

    Tables are a great way to visualize structured data. You can either use them as [blocks](https://docs.capacities.io/reference/blocks) in your objects, or turn certain tables into table objects.

    You can read more about this [here](https://docs.capacities.io/reference/basic-types/table-object).

## [Where to find your object types](https://docs.capacities.io/reference/content-types#where-to-find-your-object-types)

All your object types, both custom and basic, will be in the left sidebar.

Click on an object type, to see all objects of that type.

When you open an object type, you will be taken to the object dashboard.

## [Object Dashboards](https://docs.capacities.io/reference/content-types#object-dashboards)

Every object type has an object dashboard that you can customize. Each dashboard is unique to the object type it's connected to, so you can choose to have a different object type set up for meetings, compared to people or topics for example.

You can customize this setup by clicking on the settings icon on the right to choose which of the following sections you'd like to see *for this object type*.

- Recently opened objects

- Untagged objects

- Objects not in a collection

- Objects with no [backlinks](https://docs.capacities.io/reference/glossary#backlink)

- [Collections](https://docs.capacities.io/reference/organizational-structures#collections)

## [Filter, sort and group by](https://docs.capacities.io/reference/content-types#filter-sort-and-group-by)

You can filter to see only the items you need, sort them into a useful order, and group by a property to view them in meaningful categories.

### [Filtering](https://docs.capacities.io/reference/content-types#filtering)

If you want to find a specific group of content within a type based on its properties (e.g. all meeting notes from the past week, all books read in 2024), first open up the specific object type first (meetings, books) and then click "Filter".

Choose the property you want to look at in more depth, and the parameters you wish to set.

## [Type conversions](https://docs.capacities.io/reference/content-types#type-conversions)

If you have objects assigned to an object type that no longer fits, or you just want to change things up, you will need to know how to change object types.

Above each object's title, there is a label which tells you which object type it has. Click on that label and a dropdown menu appears. This allows you to choose which new object type you want your object to have. Click on the one you'd like to choose.

A pop up window will then come up that compares properties between each object type. Make sure your important information has somewhere to go in your new object type.

Once you're happy, click the blue button and your object and all the content you've chosen to move, will have moved!

## [FAQs](https://docs.capacities.io/reference/content-types#faqs)

- **When should I create a new object type?**

    If a group of notes has distinct properties and you can't achieve what you need with queries or property visibility alone, it's worth creating a dedicated type. See the full guide: [When to create a new object type](https://docs.capacities.io/tutorials/when-to-create-new-object-type).

- **How do I delete an object type?**

    Open the object type settings and scroll to the bottom, you will find a button to delete the object type.

- **What happens when I click "Unpin from Dashboard"?**

    This removes that section from the current object type dashboard only. For example, if you unpin "Consulting" on the People dashboard, it will no longer appear there as a tab/section.

    This is different from `Unpin from Space`, which only removes an item from the left sidebar pinned section.
