Here is the blueprint to set up your Capacities environment to perfectly match your Todoist reading and PKM goals.
Since Capacities is an object-based system, you will want to move away from standard folders and instead set up three core Object Types.

## 1. Object Type: Books

This object replaces a traditional reading log. Every time you start a new book, create it as a "Book" object.

- Suggested Icon: 📚

- Properties to Add:

- Status: Select property (Options: To Read, Reading, Completed)

    - Author: Link property (Points to the Authors object type)

    - Rating: Star rating or number scale

    - Date Finished: Date property

    - Key Takeaway: Text property (A one-sentence summary of the whole book)

## 2. Object Type: Authors

Creating an object type for authors allows you to see everything a specific person has contributed to your knowledge base.

- Suggested Icon: 👤

- Properties to Add:

- Bio / Description: Text property

    - Books: Linked collection (Automatically shows all books linked to this author)

    - Related Links: URL property (e.g., their personal website or Wikipedia page)

## 3. Object Type: Ideas / Concepts

This is the heart of your PKM system. Instead of keeping all your notes buried inside a single book page, break big theories or lessons out into individual "Idea" objects.

- Suggested Icon: 💡

- Properties to Add:

- Source: Link property (Points back to the specific Book or Daily Note where you found it)

    - Tags: Tag property (e.g., #productivity, #philosophy, #psychology)

Here is a quick guide to setting up a smart query in Capacities. This will automatically track and display every book you complete over the course of the year without any manual sorting.

## 🛠️ How to Create the "Books Completed This Year" Query

1. Create a New Query:

- Click the + button next to "Queries" in your left sidebar (or use the command palette with Ctrl + K or Cmd + K and type "Create new query").

    - Title it: 📚 Books Completed This Year.

1. Configure the Filter Criteria:
In the query settings panel, add the following rules to filter your objects:

- Object Type: Set this to Book.

    - Filter 1 (Status): Choose Status ➡️ Is ➡️ Completed.

    - Filter 2 (Date Finished): Choose Date Finished ➡️ Is after ➡️ January 1, 2026 (or use the relative setting Is within ➡️ This year if available in your current version).

1. Set the View Style:

- Change the preview layout to Gallery or Grid view. This allows you to visually scan your book covers or cards as your annual shelf grows. [1, 2]

To optimize Capacities, your Daily Note should act as a friction-free landing pad. You capture information quickly during the day, and then use links to naturally connect that data to your broader PKM network.
Here is a 3-step linking strategy designed for your reading and learning goals.

## 1. The "Log First, Structure Later" Rule

Do not worry about filling out book properties while you are actively reading. Use your Daily Note to capture fleeting thoughts, quotes, or ideas using **inline links** (`@` or ``).

Capacities automatically creates a backlink. When you open the *Atomic Habits* book object later, you will see this exact note attached to today’s date.

2. Time-Block Linkage (The Weekly Integration)

During your Sunday Todoist review, open your Daily Notes from the past week. Use Capacities' block-transformation features to turn raw text into structured assets:

- **Turn Text into Objects:** Highlight a brilliant quote or a conceptual note and convert it directly into an **Idea Object**.

- **Link the Creator:** Ensure that new Idea object links back to the original **Book Object** and the **Author Object**.

- **Contextual Links:** If an idea reminds you of a different book you read months ago, link that book directly inside the text body.

3. Track Learning Velocity via Daily Notes

Use your Daily Note to track your actual consumption metrics. This builds a timestamped history of your learning journey

    - Output: Created 2 new Idea Objects

This strategy prevents "collector's fallacy"—the habit of hoarding information without ever processing it. Your Daily Notes capture the context of when and why you learned something, while your Object Types hold the structured knowledge.
