## Key ideas (in your own words)

1. Automation engineering is broader than instrumentation or process control — it spans HAZOP studies, operator training, documentation standards, network architecture, and project management. The CAP exam tests this breadth deliberately: an automation professional who can only configure a PLC is dangerously incomplete.

2. Documentation is not optional in automation — it is a safety and regulatory requirement. P&IDs, loop diagrams, instrument lists, and logic diagrams must exist and be current for any process that handles hazardous materials. At Argenta this means GMP documentation, which has the same non-negotiable character.

3. ISA standards (ISA-5.1, ISA-88, ISA-95) are the common language of industrial automation. Learning to read and apply these standards is what separates a technician from an automation engineer. ISA-5.1 defines instrumentation symbols — if you can read a P&ID, you can read any plant.
