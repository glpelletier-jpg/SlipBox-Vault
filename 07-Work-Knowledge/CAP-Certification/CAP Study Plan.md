# CAP Study Plan

> **Goal:** Pass the ISA Certified Automation Professional (CAP) exam.  
> **Reference:** [[07-Work-Knowledge/CAP-Certification/Certified Automation Professional|CAP BOK (full domain content)]] · [[07-Work-Knowledge/CAP-Certification/CAP Practice Questions|Practice Question Log]]

---

## Exam overview

| Field | Detail |
|---|---|
| Administering body | ISA (International Society of Automation) |
| Format | 175 multiple-choice questions (155 scored, 20 unscored) |
| Duration | 3.5 hours |
| Passing score | ~68–70% |
| Cost | ~$700 ISA member · ~$1,000 non-member |
| Renewal | Every 3 years (60 PDH credits) |
| Experience required | 4 years automation engineering (min) |
| Application | Requires supervisor/employer verification |

Register at: [https://www.isa.org/certification/cap](https://www.isa.org/certification/cap)

---

## Domain 1 — Feasibility Study (8%)

- [ ] Define preliminary scope using established work practices 🆔 eEvaUk
- [ ] Perform and document cost/benefit analysis of automation alternatives 🆔 amILIi
- [ ] Conduct technical studies — machine control theories, process control theories, risk analyses 🆔 DXcrf8
- [ ] Perform justification analysis — ROI, NPV, Lean Six Sigma models 🆔 a7FXZN
- [ ] Create conceptual summary document for stakeholder approval 🆔 on0fEy

**Key knowledge areas:** Automation opportunity identification · Value analysis · Financial models (ROI, NPV) · Risk analysis

---

## Domain 2 — Definition (12%)

- [ ] Collaborate with stakeholders to identify systems, processes, operational needs 🆔 M53SLE
- [ ] Review codes, standards, documentation to create design requirements 🆔 KrPxfx
- [ ] Analyze technical approaches to establish automation strategy 🆔 u7MnsZ
- [ ] Establish basis of design — network architecture, safety, security, regulatory codes 🆔 DnceEE
- [ ] Generate cost estimates from internal/external sources 🆔 pvYbab
- [ ] Finalize basis-of-design documentation with stakeholder approval 🆔 CW4x1n

**Key knowledge areas:** ISA 88/95/62443 · Codes & standards · Interview techniques · User requirements documents

---

## Domain 3 — System Design (28%) 🔴

- [ ] Engage safety/hazard analyses — HAZOP, FMEA, PHA, fault tree (IEC 61508, ISA S84, 61511) 🆔 3ELuqy
- [ ] Establish project standards — IEC 61131 languages, ISA standards, instrument selection 🆔 OQrPct
- [ ] Create equipment specifications and instrument data sheets (ISA S20, ISA-TR20.00.01) 🆔 uVnQnN
- [ ] Specify hardware for control, safety, and security systems 🆔 PznZUV
- [ ] Select physical communication media, network architecture, protocols 🆔 icGW3A
- [ ] Develop functional description of automation solution (control scheme, alarms, HMI, reports) 🆔 XmFioj
- [ ] Establish FAT/SAT testing requirements 🆔 h152Fr
- [ ] Complete detailed design — purchase requisitions, panel designs, installation details 🆔 QTM4Da
- [ ] Plan documentation and user manuals 🆔 YSErAM

**Key knowledge areas:** ISA 5.x · P&ID interpretation · Network topology · Safety integrity levels · Electrical standards (NEC, IEC, CENELEC)

---

## Domain 4 — Development (28%) 🔴

- [ ] Develop HMI systems — alarm config, trending, historical data, human factors (ISA 101) 🆔 YR5W0i
- [ ] Develop database and reporting functions — SQL, historian/SCADA integration 🆔 WFUL5O
- [ ] Develop control configuration and programming — IEC 61131-3 languages, tag addressing 🆔 UXmndh
- [ ] Implement data transfer — communication protocols, gateways, network topology 🆔 M0DSoi
- [ ] Implement safety and security — ANSI/ISA/IEC 62443, 21 CFR Part 11, IEC 61511 🆔 A0u709
- [ ] Conduct stakeholder/peer reviews of HMI and programming 🆔 SgcEGQ
- [ ] Finalize test plan and execute FAT 🆔 bByBbe

**Key knowledge areas:** Studio 5000 / TIA Portal / DCS platforms · EtherNet/IP · Produced-consumed tags · SCADA/Historian · Security zone models

---

## Domain 5 — Deployment (16%)

- [ ] Physical inspection of installed equipment vs. design drawings 🆔 I21rGw
- [ ] Site Acceptance Testing (SAT) — loop checks, point-to-point verification 🆔 kl0CGc
- [ ] Test safety elements per test plans 🆔 aM9s15
- [ ] Test security features per test plans 🆔 lwh4oT
- [ ] Execute operational tests; adjust and tune loops 🆔 qQ1tol
- [ ] Troubleshoot installation issues using structured methodology 🆔 grVKHg
- [ ] Facilitate turnover — documentation, training, handoff to operations 🆔 kp3XUN

**Key knowledge areas:** Loop tuning · FAT/SAT methodology · Punch list processes · Change management · As-built documentation

---

## Domain 6 — Operations, Maintenance & Decommissioning (8%)

- [ ] Develop periodic inspection plans — KPIs, diagnostic tools, SPC 🆔 JmxJ4w
- [ ] Provide technical support for automation systems 🆔 TjhctG
- [ ] Conduct training needs analysis; deliver end-user training 🆔 BiT9Vr
- [ ] Monitor system performance — trending, asset management, alarming 🆔 IYdyUF
- [ ] Continuous improvement and optimization 🆔 9QLQTK
- [ ] Document lessons learned 🆔 ntZxnh
- [ ] Manage license renewals, software updates, security patches 🆔 gsLhKl
- [ ] Evaluate spare parts availability — installed base, MTBF 🆔 DoTMgi
- [ ] Predictive/preventive maintenance — backup and recovery plans 🆔 MAs9ZJ
- [ ] Facilitate management of change (MOC) 🆔 cVaD22

**Key knowledge areas:** Reliability tools · PM/PdM · MOC procedures · Disaster recovery · Version control
