# Commander — Quick Reference

> Commander adds custom action buttons to Obsidian. This vault uses two Commander surfaces for navigation and quick actions.

---

## Left Ribbon buttons

The five buttons in the left ribbon strip (far left of the app window). These are persistent navigation shortcuts — always visible regardless of what note is open.

| Button | Action | Command ID |
|---|---|---|
| 🏠 Open Home | Opens `00-Home/00-Home.md` | `homepage:open-homepage` |
| 📅 Open today's daily note | Opens or creates today's daily note | `daily-notes` |
| 📆 Open weekly note | Opens this week's review | `periodic-notes:open-weekly-note` |
| 🔍 Open search | Global vault search | `global-search:open` |
| ⚡ Open quick switcher | Fast file jump | `switcher:open` |

---

## Toolbar buttons (top of each note)

Six buttons visible at the top of every open note via the page header area. Covers navigation + quick-action needs.

| Button | Action | Command ID |
|---|---|---|
| ← Navigate back | Go back in navigation history | `app:go-back` |
| → Navigate forward | Go forward in navigation history | `app:go-forward` |
| 🏠 Open home | Opens dashboard | `homepage:open-homepage` |
| 📅 Today's daily note | Opens today's note | `daily-notes` |
| 🕸 Graph view | Opens the graph view | `graph:open` |
| ⌨ Command palette | Opens the command palette | `command-palette:open` |

---

## Tab Bar buttons

Not currently configured. The Tab Bar (row above open tabs) requires Settings → Appearance → Show tab title bar = ON. Add buttons here via Settings → Commander → Tab Bar tab if desired.

---

## How to edit buttons

1. Settings → Commander
2. Select the surface tab: **Ribbon**, **Tab Bar**, **Toolbar**, etc.
3. Click `+ Add command` to add — search by command name
4. Click an existing button to rename it or change the icon
5. Drag to reorder

---

## Configuration file

`.obsidian/plugins/cmdr/data.json`

> **Warning:** Do not edit this file directly. The format is an array (not nested object), and Obsidian must be fully closed and reopened for manual changes to take effect. Use Settings → Commander instead.

Current JSON format used by this vault (for reference only):

```json
{
  "leftRibbon": [
    {
      "id": "homepage:open-homepage",
      "icon": "lucide-house",
      "name": "Open Home",
      "mode": "any"
    }
  ],
  "titleBar": [],
  "pageHeader": [
    {
      "id": "app:go-back",
      "icon": "lucide-arrow-left",
      "name": "Navigate back",
      "mode": "any"
    }
  ]
}
```

Key rules:
- `id` must be the exact Obsidian command ID — not a UUID
- `mode` must be `"any"` for the button to render on desktop
- Tab Bar (`titleBar`) buttons additionally require Settings → Appearance → Show tab title bar = ON

---

## Troubleshooting

**Button shows in Commander settings but doesn't appear**
All three must be true: (1) `id` is a valid command ID, (2) `mode: "any"` is present, (3) for Tab Bar, Show tab title bar is ON.

**Button appears but click does nothing**
The plugin providing that command was disabled, or the QuickAdd choice was deleted. Remove the button and re-add it — search again by name.

**"No commands here!" in Commander settings**
Entries in `data.json` have invalid command IDs. Open Settings → Commander → use the UI to add commands, do not edit JSON directly.
