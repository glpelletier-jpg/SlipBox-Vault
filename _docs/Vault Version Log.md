## Versioning convention

| Change type | Version bump | Examples |
|---|---|---|
| New folder, major new workflow, plugin added | Minor (x.**Y**.0) | Adding 08-Work-Notes, new procedure doc, new base panel |
| Bug fix, content addition, hotkey change | Patch (x.y.**Z**) | Fixing broken link, adding notes, updating frontmatter |
| Vault restructure, migration, major overhaul | Major (**X**.0.0) | Renaming all folders, switching review system |

Update the version string in `00-Home/00-Home.md` (bottom of file) when incrementing.
